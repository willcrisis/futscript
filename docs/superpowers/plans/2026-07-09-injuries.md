# Richer Injuries Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make injuries drop a player's level (scaled by severity), recover it week by week toward a permanently-shaved ceiling, and permanently raise the player's re-injury chance.

**Architecture:** Two new `Player` fields — `peakLevel` (recovery ceiling) and `injuryCount` (proneness) — threaded through a v8→v9 migration and every player-creation site. The injury branch, the weekly-update recovery, and age decline all move `level`/`peakLevel` together; the match engine weights the injury victim by proneness.

**Tech Stack:** TypeScript, React, Vitest.

## Global Constraints

- Engine stays pure (no React/DOM/i18n imports; randomness threaded through `rand`; money integer dollars).
- Semantic design tokens only; reuse the `src/ui/` kit; new i18n keys go in both `en.ts` and `pt.ts` (compile-checked).
- Save version bumps **8 → 9**; `migrateV8` adds `peakLevel = level`, `injuryCount = 0` to every player.
- `level <= peakLevel` is an invariant everywhere.
- Balance numbers are `ponytail:`-tagged constants: `DROP_PER_WEEK = 1.5`, `PERMA_DIVISOR = 3`, `RECOVER_STEP = 1`, `PRONENESS_WEIGHT = 0.15`, `PRONE_THRESHOLD = 3`.
- Typecheck `npx tsc -b --force`; tests `npm test` / `npx vitest run <file>`.
